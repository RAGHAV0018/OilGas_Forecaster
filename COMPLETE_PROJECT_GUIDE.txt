SECTION 1: PROJECT OVERVIEW
================================================================================

What does this project do?
--------------------------
This project predicts future oil and gas prices in India by combining TWO
types of AI:

ENGINE 1 - NLP (Natural Language Processing):
  Reads news headlines (like "OPEC cuts oil production") and understands
  whether they will cause oil prices to go UP, DOWN, or stay STABLE.
  Uses a pre-trained AI model called FinBERT.

ENGINE 2 - LSTM (Long Short-Term Memory Neural Network):
  Looks at the last 30 days of historical oil prices and mathematically
  predicts what the price will be for the next 30 days.

ENGINE 3 - Dashboard (Streamlit Web App):
  A website that runs on your computer where both engines are displayed
  side-by-side so evaluators can interact with them visually.

Phase 1 Status: Both engines work INDEPENDENTLY (side-by-side).
Phase 2 Goal:   The NLP score will be FED INTO the LSTM to alter predictions.

FILE 2: src/nlp_scorer.py  (ENGINE 1 - The Text Brain)
------------------------------------------------------------------------

PURPOSE: Takes a news headline as input, analyzes it, and outputs whether
oil prices will go UP, DOWN, or stay STABLE.

LINE-BY-LINE EXPLANATION:

  Lines 1-3: IMPORTS
  -------------------
  import warnings              -> Hides unnecessary warning messages
  warnings.filterwarnings('ignore')
  from transformers import pipeline  -> Imports HuggingFace's "pipeline"
                                        function which is the easiest way
                                        to load and use a pre-trained AI model

  Lines 5-12: CLASS INITIALIZATION
  ---------------------------------
  class GeopoliticalRiskScorer:
      def __init__(self):
          self.nlp_pipeline = pipeline("text-classification",
                                       model="ProsusAI/finbert")

  What is happening here?
  - We create a class called GeopoliticalRiskScorer.
  - When you create an object of this class, it automatically downloads and
    loads the FinBERT model from the internet (first time only, ~400MB).
  - "pipeline" is a HuggingFace shortcut. Instead of writing 50 lines of
    code to load a model, tokenize text, and run inference, pipeline() does
    it all in one line.
  - "text-classification" tells HuggingFace we want to CLASSIFY text into
    categories (positive/negative/neutral).
  - "ProsusAI/finbert" is the specific model. FinBERT was trained on 50,000+
    financial news articles, so it understands financial language better than
    a generic sentiment model.

  Lines 14-41: THE ANALYSIS FUNCTION
  ------------------------------------
  def analyze_headline(self, headline):
      result = self.nlp_pipeline(headline)[0]
      sentiment = result['label']      # e.g., "negative"
      confidence = result['score']     # e.g., 0.9543

  What is happening here?
  - We pass the headline text into the FinBERT pipeline.
  - It returns a dictionary with:
      'label': one of "positive", "negative", or "neutral"
      'score': how confident the model is (0.0 to 1.0)

  THE TRANSLATION LOGIC (Lines 25-33):
  - This is the KEY INNOVATION of our project.
  - In general sentiment analysis, "OPEC cuts production" = NEGATIVE sentiment.
  - But in OIL MARKETS, less supply = higher prices = BULLISH for oil.
  - So we REVERSE the logic:
      negative sentiment -> "HIGH RISK / SUPPLY SHOCK" -> Price goes UP
      positive sentiment -> "STABLE / OVERSUPPLY"      -> Price goes DOWN
      neutral sentiment  -> "NEUTRAL"                   -> Price STABLE

  Lines 43-59: TEST DEMONSTRATION
  ---------------------------------
  Three sample headlines are tested when you run the file directly:
  1. "OPEC announces unexpected cut..." -> Should flag HIGH RISK, price UP
  2. "Peace treaty signed..."           -> Should flag STABLE, price DOWN
  3. "India boosts EV subsidies..."     -> Should flag NEUTRAL/STABLE

FILE 3: src/price_predictor.py  (ENGINE 2 - The Math Brain)
------------------------------------------------------------------------

PURPOSE: Uses past oil price data to mathematically predict future prices
using an LSTM (Long Short-Term Memory) neural network.

This file has THREE main components:

COMPONENT A: generate_mock_oil_data() [Lines 14-28]
----------------------------------------------------
  def generate_mock_oil_data(days=1000):
      np.random.seed(42)
      base_price = 75.0

  What is happening here?
  - Since we don't have real PPAC/OPEC CSV files yet (Phase 2), this
    function GENERATES fake but realistic oil price data.
  - It starts at $75/barrel and uses a "random walk" algorithm:
      Each day, the price moves up or down randomly by a small amount.
  - np.random.seed(42) ensures the same fake data is generated every time
    you run the script (for reproducibility during demos).
  - max(20.0, new_price) ensures prices never go below $20 (unrealistic).
  - It creates a pandas DataFrame with columns: Date and Price.

COMPONENT B: OilPriceLSTM [Lines 33-54]
-----------------------------------------
  class OilPriceLSTM(nn.Module):
      def __init__(self, input_size=1, hidden_size=50, num_layers=1):
          self.lstm = nn.LSTM(input_size, hidden_size, num_layers,
                              batch_first=True)
          self.fc = nn.Linear(hidden_size, 1)

  What is an LSTM?
  - LSTM stands for Long Short-Term Memory. It is a special type of
    Recurrent Neural Network (RNN).
  - Normal neural networks see data as independent points. But oil prices
    are SEQUENTIAL — today's price depends on yesterday, which depends on
    the day before, and so on.
  - LSTMs have "memory cells" that can remember patterns from the past.
    For example, if prices went up for 5 days straight, the LSTM recognizes
    this upward momentum and factors it into its prediction.

  Architecture of our LSTM:
  - input_size=1: We feed ONE number per timestep (the price).
  - hidden_size=50: The LSTM has 50 internal "neurons" that process data.
    More neurons = more capacity to learn complex patterns, but also slower.
  - num_layers=1: We use a single LSTM layer (keeping it simple for Phase 1).
  - self.fc = nn.Linear(hidden_size, 1): A "Fully Connected" layer that
    takes the 50-neuron output from the LSTM and compresses it down to
    a single number: the predicted price.

  The forward() function [Lines 44-53]:
  - h0 and c0 are the "hidden state" and "cell state" — initialized to
    zeros. These are the LSTM's memory banks.
  - self.lstm(x, (h0, c0)) runs the input through the LSTM.
  - out[:, -1, :] takes ONLY the output from the LAST timestep (day 30),
    because we only care about predicting the NEXT day.
  - self.fc(out) converts the LSTM output into a single price prediction.

COMPONENT C: PricePredictorEngine [Lines 59-113]
--------------------------------------------------
  This class manages the entire workflow: data prep, training, prediction.

  prepare_data() [Lines 67-79]:
  - MinMaxScaler: Neural networks work best when input numbers are small
    (between -1 and 1). Oil prices like $85.50 are too large.
    MinMaxScaler transforms all prices using this formula:
        scaled = (price - min_price) / (max_price - min_price)
    This maps everything to the range [-1, 1].
  - Sequence Creation: We create "sliding windows" of 30 days.
    Example: Days 1-30 -> predict Day 31
             Days 2-31 -> predict Day 32
             Days 3-32 -> predict Day 33 ... and so on.
    This gives the LSTM many training examples from a single dataset.

  train() [Lines 81-97]:
  - Epochs: One epoch = one complete pass through all the training data.
    We run 50 epochs, meaning the model sees the data 50 times.
  - optimizer.zero_grad(): Clears old gradient calculations.
  - loss = criterion(outputs, y): Calculates Mean Squared Error (MSE).
    MSE = average of (predicted_price - actual_price)^2
    The goal is to minimize this number.
  - loss.backward(): Computes gradients (how much to adjust each weight).
  - optimizer.step(): Actually adjusts the weights using Adam optimizer.
    Adam adapts the learning rate automatically for each parameter.

  predict_future() [Lines 99-113]:
  - After training, we feed the model the LAST 30 days of known data.
  - It predicts Day 31. We append that prediction to the sequence.
  - Now we feed Days 2-31 (including the predicted Day 31) to predict Day 32.
  - This continues for 30 iterations to get a full month of predictions.
  - scaler.inverse_transform() converts the scaled predictions (-1 to 1)
    back to real dollar values.

FILE 4: src/dashboard.py  (ENGINE 3 - The Visual Interface)
------------------------------------------------------------------------

PURPOSE: Creates a beautiful, interactive web page that combines both
engines into a single visual experience.

STREAMLIT BASICS:
  - Streamlit is a Python library that converts Python scripts into websites.
  - You run it with: streamlit run dashboard.py
  - It opens a browser tab at http://localhost:8501
  - Every time you interact with a widget (button, text box), Streamlit
    re-runs the ENTIRE script from top to bottom.

LINE-BY-LINE BREAKDOWN:

  Lines 1-9: IMPORTS
  -------------------
  - streamlit (st): The web framework.
  - plotly.graph_objects (go): For creating the interactive price chart.
  - We import our OWN files: GeopoliticalRiskScorer and PricePredictorEngine.

  Lines 14-19: PAGE CONFIGURATION
  ---------------------------------
  st.set_page_config(
      page_title="Oil & Gas AI Forecaster",   -> Browser tab title
      page_icon="🛢️",                         -> Browser tab icon
      layout="wide",                           -> Uses full screen width
  )

  Lines 26-48: MODEL LOADING WITH CACHING
  -----------------------------------------
  @st.cache_resource
  def load_nlp_model():
      return GeopoliticalRiskScorer()

  Why @st.cache_resource?
  - Remember: Streamlit re-runs the entire script on every interaction.
  - Without caching, the 400MB FinBERT model would reload every time you
    click "Analyze". That would take 30+ seconds per click.
  - @st.cache_resource tells Streamlit: "Load this ONCE, store it in
    memory, and reuse it for all future interactions."
  - Same logic applies to the LSTM training — we train ONCE and cache.

  Lines 53-78: LEFT COLUMN - NLP RISK SCORER
  --------------------------------------------
  col1, col2 = st.columns([1, 1])   -> Splits page into 2 equal columns

  What the user sees:
  - A header: "📰 Geopolitical Risk Scorer"
  - A text box with a sample headline pre-filled
  - An "Analyze Impact" button
  - After clicking, three colored boxes appear:
      BLUE box:  Shows the raw sentiment (POSITIVE/NEGATIVE/NEUTRAL)
      RED box:   Shows "HIGH RISK / SUPPLY SHOCK" (if negative news)
      YELLOW box: Shows predicted price trend (UPWARDS/DOWNWARDS/STABLE)

  Lines 80-122: RIGHT COLUMN - LSTM PRICE CHART
  ------------------------------------------------
  What the user sees:
  - A header: "📈 LSTM Price Prediction"
  - An interactive Plotly chart with:
      BLUE solid line:  Last 30 days of historical prices
      RED dashed line:  Next 30 days of LSTM predictions
  - The chart is interactive: you can zoom, pan, hover over points to see
    exact values, and toggle lines on/off.

  Lines 127-131: FOOTER
  -----------------------
  A note explaining that in Phase 2, these two engines will be integrated.

SECTION 5: HOW EVERYTHING CONNECTS
================================================================================

  dashboard.py ──imports──> nlp_scorer.py (GeopoliticalRiskScorer class)
       │
       └────imports──> price_predictor.py (PricePredictorEngine class,
                                            generate_mock_oil_data function)

  When you run "streamlit run src/dashboard.py":
  1. Streamlit starts a local web server on port 8501.
  2. dashboard.py imports and initializes both AI models.
  3. The NLP model downloads FinBERT from HuggingFace (first run only).
  4. The LSTM model generates fake price data and trains on it.
  5. Both results are cached so subsequent interactions are instant.
  6. The web page renders with both engines side-by-side.

  PHASE 2 CONNECTION (Future):
  Currently:  NLP Score and LSTM work separately.
  Phase 2:    NLP Score becomes an INPUT FEATURE to the LSTM.
              If NLP detects "HIGH RISK", the LSTM adjusts its prediction
              upward. This is the "Multi-Signal" fusion.

SECTION 7: KEY TERMS GLOSSARY
================================================================================

NLP (Natural Language Processing):
  Teaching computers to understand human language (text).

FinBERT:
  A version of Google's BERT model, retrained specifically on financial
  news articles. Built by Prosus AI. Available free on HuggingFace.

LSTM (Long Short-Term Memory):
  A neural network designed for sequential/time-series data. It has
  internal "memory" that helps it remember past patterns.

PyTorch:
  An open-source ML framework by Meta (Facebook). We use it to build
  and train the LSTM. Alternative: TensorFlow by Google.

MinMaxScaler:
  A tool from scikit-learn that normalizes data to a fixed range
  (we use -1 to 1). Formula: scaled = (x - min) / (max - min)

Epoch:
  One complete pass of the training data through the neural network.
  More epochs = the model learns more, but too many = overfitting.

Loss (MSE - Mean Squared Error):
  The error metric. MSE = average of (prediction - actual)^2.
  Lower loss = better predictions.

Adam Optimizer:
  An algorithm that adjusts the neural network's internal weights
  during training. It adapts the learning speed automatically.

Streamlit:
  A Python library that turns Python scripts into interactive web apps.
  Used by data scientists worldwide to build ML dashboards quickly.

Plotly:
  A graphing library that creates interactive, zoomable, hoverable
  charts. Better than matplotlib for web dashboards.

Pipeline (HuggingFace):
  A one-line function that loads a pre-trained model and runs
  inference. Simplifies complex NLP tasks to a single function call.

RAG (Retrieval Augmented Generation):
  A technique where an AI chatbot retrieves relevant documents from
  a database before generating an answer. Used in Phase 2.

Backtest:
  Testing a prediction model against historical events to see if it
  would have predicted them correctly (e.g., COVID crash).
