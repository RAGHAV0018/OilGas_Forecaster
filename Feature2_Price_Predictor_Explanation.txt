# Feature 2 Explanation: Baseline Time-Series Price Predictor

## What is this feature?
This is the second core component of Phase 1: the **LSTM Numerical Forecasting Model** (`src/price_predictor.py`). 

While the first feature (`nlp_scorer.py`) handles text and news, this feature handles **pure numbers and historical prices**. This proves to the evaluators that you have a working mathematical model capable of finding patterns in past data to predict future prices.

## How the Code Works (Step-by-Step for your understanding)

### 1. Data Generation (`generate_mock_oil_data`)
Because we haven't connected to the live Indian government PPAC databases yet (that happens in Phase 2), this function uses Python to mathematically generate **fake historical oil prices**. It creates realistic data that mimics how oil prices fluctuate daily (using something called a "random walk with volatility"). This ensures the code is runnable immediately for your Phase 1 demo without needing external CSV files.

### 2. The AI Brain (`OilPriceLSTM`)
This is the actual Machine Learning architecture. We use an **LSTM (Long Short-Term Memory)** network built with PyTorch. 
LSTMs are specifically designed for Time-Series data (like stock prices or oil prices) because they have "memory". They don't just look at today's price; they remember the pattern of the last 30 days to make a decision about day 31.

### 3. The Engine (`PricePredictorEngine`)
This class acts as the manager. It does three things:
- **`prepare_data`**: Neural networks hate large numbers (like $85.00). This function uses a `MinMaxScaler` to shrink all prices down to a scale between -1 and 1 so the AI can learn faster. It also groups the data into 30-day chunks.
- **`train`**: This function feeds the data into the LSTM. It runs it through 50 times (epochs), adjusting its internal weights to reduce the "Loss" (the error rate between its prediction and reality). 
- **`predict_future`**: Once trained, we feed it the *most recent* 30 days, and it spits out the predicted prices for the next 30 days.

## How it fits into the overall project:
Right now, this model predicts prices purely based on *past prices*. 
In **Phase 2**, we will take the "Geopolitical Risk Score" generated by `nlp_scorer.py`, and we will inject it into this LSTM network. That is when the project becomes the hybrid "Multi-Signal AI Framework" you presented in your PPT!
